# FRRG 底层抓取闭环任务计划书

## 1. 任务整体目的

本任务的目标是在现有 `vlbiman_sa` 单臂流程中新增一层底层抓取运动闭环模块，作为粗接近之后的原子任务执行层，代替原流程中对抓取末段和接触段的 `inv` 不变轨迹重放。

现有 VLBiMan 单臂流程已经具备 one-shot 采集、技能切分、视觉锚点估计、位姿适配、轨迹生成和粗接近执行能力。本任务不重复建设这些能力，而是在粗接近到达预抓壳层后，接管最后几厘米的入爪、闭爪、稳持和小提起验证，形成可观测、可调参、可回退、可逐步学习的抓取闭环。

最终目标是把抓取从“录制轨迹直接重放”改为：

```text
VLBiMan 粗接近输出预抓位姿
  -> FRRG 底层抓取闭环
    -> Approach-Handoff 粗接近交接
    -> Capture-Build 入爪构型建立
    -> Close-Hold 闭爪稳持确认
    -> Lift-Test 小提起验证
    -> 成功 / 恢复 / 失败上报
```

本任务首版采用“固定流程 + 示教参数化 + 局部残差修正”的工程路线：

- 抓取阶段状态机固定，不交给学习模型决定主流程。
- 阶段参数可以从示教中抽取，并支持后续用低维回归器生成。
- phase-2 和 phase-4 的名义短程动作可以由 DMP 或轻量轨迹模板表示。
- phase-2 和 phase-3 的最后几厘米误差用受限残差策略补偿。
- 安全边界、速度上限、闭爪上限、视觉丢失停止、最大重试次数全部显式写死。

## 2. 当前接入位置

FRRG 接在现有项目的以下位置：

```text
现有流程：
one-shot demo
  -> T3 skill_bank
  -> T4 vision anchor
  -> T5 pose adaptation
  -> T6 trajectory generation
  -> T7 robot execution

调整后：
one-shot demo
  -> T3 skill_bank
  -> T4 vision anchor
  -> T5 pose adaptation
  -> T6 coarse approach trajectory
  -> FRRG grasp closed loop
  -> T7 robot execution / report
```

具体含义：

- `T6` 仍负责生成到预抓壳层的粗接近轨迹。
- 原来 `inv` 段中“靠近后直接闭爪、搬运动作直接复用”的部分不再作为抓取成功的依据。
- FRRG 从当前末端状态、目标局部坐标系、手眼相机局部误差和夹爪状态开始运行。
- FRRG 输出最终抓取状态：`success`、`recoverable_failure`、`terminal_failure`，并产生日志和诊断文件。

## 3. 任务实施方案

### 3.1 第一阶段原则

第一阶段不做端到端学习，不做 RL，不依赖大模型在线控制。先完成纯工程闭环：

1. 固定 4 状态机。
2. 显式状态变量和阶段 guard。
3. 手写安全边界。
4. 可 dry-run 的执行器。
5. 可用离线样本和 mock 状态跑通单元测试。
6. 真机只在 dry-run 和分阶段验收通过后执行。

### 3.2 第二阶段原则

第二阶段引入示教参数化：

1. 从已有 one-shot 或新采集示教中提取末段抓取参数。
2. 建立 `theta` 阶段参数格式。
3. 用规则或 kNN/随机森林等低维模型生成参数。
4. 保持安全边界不学习。

### 3.3 第三阶段原则

第三阶段引入受限残差：

1. 只在 `Capture-Build` 和 `Close-Hold` 两个阶段启用。
2. 残差输出被硬裁剪，例如 `dx/dy <= 4mm`、`dz <= 3mm`、`dyaw <= 5deg`。
3. 先做 BC，从示教动作减去名义动作得到残差标签。
4. 后续如需要再做仿真或安全边界内 residual RL。

## 4. 任务系统架构

### 4.1 在线数据流

```text
run_visual_closed_loop_on_robot.py / run_execute_t6_on_robot.py
  -> 粗接近执行到 T_pre
  -> FRRGHandoffState
    -> FrameManager: base/camera/tool/object 坐标转换
    -> GraspStateObserver: 手眼视觉 + 夹爪 + 末端位姿状态
    -> PhaseParameterProvider: 生成 theta
    -> NominalPrimitiveRunner: phase-2 / phase-4 名义动作
    -> ResidualPolicy: phase-2 / phase-3 局部补偿
    -> PhaseGuardEvaluator: 阶段切换和失败判定
    -> SafetyLimiter: 动作限幅、安全停止
    -> GraspClosedLoopController
  -> RobotCommandAdapter
  -> CjjArm.send_action
  -> GraspExecutionReport
```

### 4.2 状态机

```text
HANDOFF
  -> CAPTURE_BUILD
    -> CLOSE_HOLD
      -> LIFT_TEST
        -> SUCCESS

任一阶段：
  -> RECOVERY
  -> FAILURE
```

各阶段定义：

- `HANDOFF`：确认粗接近已到达预抓壳层，手眼相机能看到目标，夹爪已打开到安全宽度。
- `CAPTURE_BUILD`：末端在物体局部坐标系下执行最后几厘米推进和微对位，直到目标进入有效两指通道。
- `CLOSE_HOLD`：闭合夹爪并用夹爪开口、电流替代量、视觉漂移判断是否稳持。
- `LIFT_TEST`：小幅上提并观察短窗口内是否滑移。
- `RECOVERY`：微退、半开夹爪、重新估计目标局部误差，并最多重试 `N` 次。
- `FAILURE`：停止输出动作，上报失败原因和最后状态快照。

### 4.3 核心状态变量

每个控制周期维护：

```text
s_t = {
  top_features,
  wrist_features,
  object_frame,
  ee_pose_base,
  ee_pose_object,
  joint_positions,
  gripper_position,
  gripper_current_proxy,
  phase,
  primitive_progress,
  retry_count,
  timestamps
}
```

首版必须支持的手眼局部误差：

- `e_lat`：目标相对夹爪中线的横向偏差。
- `e_dep`：目标相对夹爪推进方向的深度差。
- `e_sym`：左右边缘或左右可见间隙差。
- `e_ang`：目标主轴与夹爪闭合方向夹角。
- `occ_corridor`：两指通道占据率或目标进入通道置信度。
- `drift_obj`：连续帧目标相对夹爪漂移量。

### 4.4 阶段参数 theta

首版参数结构：

```text
theta = {
  pregrasp_offset_m,
  corridor_half_width_m,
  advance_speed_mps,
  angular_speed_radps,
  preclose_distance_m,
  preclose_pause_s,
  close_speed_raw_per_s,
  settle_time_s,
  lift_height_m,
  slip_threshold_m,
  guard_stable_frames,
  max_retry_count
}
```

### 4.5 安全边界

必须显式实现：

- 最大平移步长。
- 最大旋转步长。
- 最大单周期关节变化。
- 最大闭爪速度。
- 最大闭爪位置或力/电流替代阈值。
- 视觉丢失停止。
- IK 失败停止。
- 目标跳变停止。
- 最大重试次数。
- dry-run 默认开启，真机执行需要显式关闭 dry-run。

### T5 阶段 guard 与成功/失败判定

**实现目标**

实现每个阶段的显式 guard，使系统能够判断何时切换、何时恢复、何时失败。

**具体实施方案**

- 新建 `grasp/phase_guards.py`。
- `HANDOFF -> CAPTURE_BUILD` 条件：
  - 粗接近完成。
  - 目标可见。
  - 夹爪打开。
  - 当前末端距离预抓位姿小于阈值。
- `CAPTURE_BUILD -> CLOSE_HOLD` 条件：
  - `abs(e_lat) < corridor_half_width_m`
  - `abs(e_ang) < tau_ang`
  - `occ_corridor > tau_occ`
  - 连续满足 `guard_stable_frames` 帧。
- `CLOSE_HOLD -> LIFT_TEST` 条件：
  - 夹爪开口处于合理范围。
  - 电流或接触替代量进入接触区间。
  - 视觉漂移小于阈值。
- `LIFT_TEST -> SUCCESS` 条件：
  - 小提起后 `drift_obj < slip_threshold_m`。
  - 未出现接触丢失。
  - 目标没有异常脱离视野。

## 9. 首版完成定义

FRRG 首版认为完成，需要同时满足：

1. 粗接近后的抓取不再依赖原 `inv` 不变末段重放。
2. `HANDOFF -> CAPTURE_BUILD -> CLOSE_HOLD -> LIFT_TEST` 状态机可 dry-run 完整跑通。
3. 每一步动作都经过安全限幅。
4. 每个失败都有明确原因和报告文件。
5. 可以从 T6 输出启动 FRRG。
6. 可以从 one-shot 示教提取 theta 样本。
7. 真机 dry-run 能生成报告且不发动作。
8. 分阶段真机执行能够按 `--stop-after-phase` 停止。
9. 小规模评测报告能对比原 inv 重放基线。
