本任务目标是在现有 `lerobot_vlbiman` 工程中新增一套独立的 `inv RGB 视觉伺服抓取架构`，用于替换或增强原有 T6 粗接近之后的 inv 抓取段。

```text

```

本任务当前新增重点包括：

```text
1. GroundingDINO 单独作为目标检测阶段
2. GroundingDINO 可参考本机已有路径：
   /home/cjj/ViT-VS/GD_DINOv2_Sim/GroundingDINO

3. GroundingDINO 验收必须使用已有数据：
   /home/cjj/lerobot_vlbiman/outputs/vlbiman_sa/original_flow/pure_sim_original_20260427T072327Z

4. GroundingDINO 必须对手上相机第 75 帧图像进行 yellow ball 检测
5. yellow ball 检测成功后，GroundingDINO 阶段才算完成

6. SAM2 单独作为实时跟踪与 mask 提取阶段
7. SAM2 可参考工程内已有实现
8. SAM2 必须基于 GroundingDINO 第 75 帧检测结果初始化
9. SAM2 必须对第 75 到第 100 帧的 yellow ball 持续提取 mask
10. SAM2 阶段完成时必须输出一组 mask 图片供人工检查

11. RGB 视觉伺服控制模块完成后，必须进行第 75 帧到第 100 帧目标对齐验证
12. 验证时从第 75 帧仿真场景开始，完整调用：
    GroundingDINO → SAM2 → RGB 视觉伺服控制器 → 仿真机械臂执行后端

13. 控制目标为：
    让当前手上相机中的 yellow ball mask 对齐第 100 帧手上相机的 yellow ball mask

14. 只有机械臂成功伺服到 mask 对齐目标，阶段 7 才算完成
```

整体流程如下：

```text
T6 粗接近完成
  ↓
进入 inv 抓取段
  ↓
读取目标视觉描述或目标参考状态
  ↓
GroundingDINO 检测 yellow ball
  ↓
用第 75 帧检测框初始化 SAM2
  ↓
SAM2 对第 75 到第 100 帧持续提取 yellow ball mask
  ↓
取第 100 帧 hand/wrist camera 的 yellow ball mask 作为视觉伺服目标
  ↓
从第 75 帧仿真场景启动闭环
  ↓
当前相机图像 → GroundingDINO → SAM2 → 当前 mask
  ↓
当前 mask 与第 100 帧目标 mask 计算误差
  ↓
RGB 视觉伺服控制器计算机械臂小步动作
  ↓
仿真执行后端执行动作
  ↓
循环直到当前 mask 对齐第 100 帧目标 mask
  ↓
达到收敛条件后闭合夹爪
  ↓
返回原 VLBiMan 后续流程
```

## 3、总体架构分层

整体架构分为 6 层：

```text
入口层
  ↓
任务执行层
  ↓
感知层
  ↓
目标状态层
  ↓
视觉伺服控制层
  ↓
执行后端层
```

各层职责：

```text
入口层：
  解析参数、加载配置、创建执行器。

任务执行层：
  串联检测、跟踪、控制、安全检查、动作执行和闭爪。

感知层：
  负责 GroundingDINO 检测和 SAM2 实时跟踪。

目标状态层：
  从目标 mask 或目标状态中生成视觉伺服目标。

视觉伺服控制层：
  计算当前 mask 与目标 mask 的视觉误差，并生成小步控制量。

执行后端层：
  连接真实机器人、仿真器或离线执行接口。
```

### 7.1 rgb_servo_controller.py

模块职责：

```text
接收当前 mask 状态
接收第 100 帧目标 mask 状态
计算中心误差
计算面积误差
可选计算 mask 重叠误差
输出相机坐标系下小步动作 delta_cam
输出 6DoF 末端位姿增量 delta_pose
判断是否收敛
```

输入结构：

```python
{
    "current_center_uv": [u, v],
    "current_area_ratio": float,
    "current_mask": np.ndarray | None,
    "target_center_uv": [u_star, v_star],
    "target_area_ratio": float,
    "target_mask": np.ndarray | None,
    "image_width": int,
    "image_height": int,
}
```

误差定义：

```text
e_u = (u - u*) / image_width
e_v = (v - v*) / image_height
e_a = log((a* + eps) / (a + eps))
```

mask 对齐指标：

```text
mask_iou = intersection(current_mask, target_mask) / union(current_mask, target_mask)
```

其中：

```text
current_mask：
  当前仿真闭环中，手上相机看到的 yellow ball mask

target_mask：
  阶段 6 生成的第 100 帧手上相机 yellow ball mask，即：
  outputs/vlbiman_sa/inv_rgb_servo/check_sam2_75_100/mask_000100.png
```

控制输出：

```text
dx_cam = -k_u * e_u
dy_cam = -k_v * e_v
dz_cam =  k_a * e_a
rotation_rpy = [0, 0, 0]  # 临时占位，接口保留 6DoF
```

输出结构：

```python
{
    "delta_cam": [dx, dy, dz],
    "delta_pose": {
        "translation": [dx, dy, dz],
        "rotation_rpy": [droll, dpitch, dyaw]
    },
    "error": {
        "e_u": float,
        "e_v": float,
        "e_a": float,
        "error_norm": float,
        "mask_iou": float | None,
        "mask_iou_error": float | None,
    },
    "converged": bool,
}
```

收敛条件：

```text
必须同时满足：

abs(e_u) <= center_tol_u
abs(e_v) <= center_tol_v
abs(e_a) <= area_tol
mask_iou >= mask_iou_tol
连续稳定帧数量 >= stable_frames
```

建议默认阈值：

```text
center_tol_u = 0.035
center_tol_v = 0.035
area_tol = 0.12
mask_iou_tol = 0.70
stable_frames = 5
max_steps = 120
```

```text
后端执行失败：停止执行，不闭爪
```

# 5、错误审查流程

## 5.1 审查优先级

```text
第一优先级：输入参数检查
第二优先级：配置检查
第三优先级：inv_servo 新增模块内部修复
第四优先级：模型调用参数修复
第五优先级：仿真执行后端适配修复
第六优先级：真实机器人执行后端适配修复
第七优先级：T6 接入层修复
最低优先级：修改原有底层系统
```

最低优先级包括：

```text
修改 handeye
修改 record
修改 skill_build
修改 vision/vlm_segmentor.py
修改 pose_adaptation
修改 trajectory_generation / T6 核心逻辑
修改原机器人控制底层
修改原有 SAM2 数据链
```

原则：

```text
能在 inv_servo 内解决，不修改原模块
能在配置中解决，不修改代码
能在适配层解决，不改底层
触动原系统必须单独说明原因
```
